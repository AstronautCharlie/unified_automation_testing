  Plan: Estimating Time Each Individual Spends in Each Posture

  What the data provides

  - 644,635 rows sampled at ~30 Hz (one row every ~33 ms)
  - 4 subjects (S1вҖ“S4), identified by the subject column, each with 5 ADL
  recordings in the recording column
  - No locomotion labels вҖ” posture must be inferred from raw sensor data
  - Key sensors available (with <7% null rates):
    - InertialMeasurementUnit_BACK_acc[X/Y/Z] вҖ” back-mounted IMU accelerometers
    - InertialMeasurementUnit_[L/R]-SHOE_AngVelBodyFrame[X/Y/Z] вҖ” shoe angular
  velocity
    - Accelerometer_RKN^_acc[X/Y/Z] вҖ” right knee accelerometers (~7% null)

  ---
  Step 1: Handle missing values

  For each key sensor column, forward-fill then backward-fill missing values
  within each recording. This preserves temporal continuity without fabricating
  data. Flag rows that were imputed (in case null rate is itself a useful
  signal).

  ---
  Step 2: Feature engineering

  Compute three derived signals per row:

  A. Back tilt (detects lying)
  back_tilt = atan2(sqrt(BACK_accYВІ + BACK_accZВІ), -BACK_accX)
  The data shows BACK_accX mean вүҲ вҲ’862 milli-g, close to gravity (1000 milli-g),
   indicating the X-axis captures the vertical gravity component when upright.
  When lying down, gravity shifts to another axis, so |BACK_accX| drops
  significantly.

  B. Shoe angular velocity magnitude (detects walking)
  shoe_angvel = sqrt(mean([L-SHOE, R-SHOE]_AngVelBodyFrame[X/Y/Z]ВІ))
  Use a 3-second rolling window (вүҲ90 samples at 30 Hz). Walking produces
  sustained, high shoe angular velocity; standing and sitting produce near-zero
  values.

  C. Knee gravity component (distinguishes sitting from standing)
  knee_gravity = RKN^_accY
  The data shows RKN^_accY median вүҲ 946 milli-g when leg is straight (standing).
   When seated, the knee bends ~90В° and this component drops substantially.

  ---
  Step 3: Classify each timestep (priority order)

  Apply these rules in order вҖ” the first matching rule wins:

  1. Lying: |BACK_accX| < ~500 milli-g (back horizontal, gravity no longer along
   X-axis)
  2. Walking: rolling mean of shoe_angvel > threshold (~2000 milli-gВ·sвҒ»В№) over a
   3-second window
  3. Sitting: knee_gravity < ~500 milli-g (knee bent at ~90В°)
  4. Standing: all other cases

  Thresholds should be tuned by inspecting the distribution of each feature; the
   values above are informed by the data stats observed (e.g., RKN^_accY 75th 
  pctile = 1027, 25th pctile = 491).

  ---
  Step 4: Temporal smoothing

  Apply a majority-vote filter over a 1-second sliding window (30 samples) to
  remove brief, spurious transitions caused by sensor noise or momentary
  movement. Additionally, enforce a minimum duration of 5 seconds for "lying" (a
   person briefly lying flat for <5 seconds is more likely noise).

  ---
  Step 5: Compute time per posture per subject

  For each subject:
  - Group all rows by predicted posture class
  - Count rows per class вҶ’ multiply by 33 ms/row to get time in seconds
  - Sum across all recordings for that subject
  - Optionally break down by recording (ADL1вҖ“ADL5) to see within-session
  variation
